What is Kinematics?

Was ist Kinematik?

The study of motion — position, velocity, orientation — without considering the forces that cause it.

Die Lehre von der Bewegung — Position, Geschwindigkeit, Orientierung — ohne Berücksichtigung der Kräfte, die sie verursachen.

  • Where is the end-effector, given the joint values?
  • What joint values are needed to reach a desired end-effector pose?
  • Wo befindet sich der Endeffektor bei gegebenen Gelenkwerten?
  • Welche Gelenkwerte sind nötig, um eine gewünschte Endeffektor-Pose zu erreichen?

Relevant for every robot arm, 3D printer, CNC machine, and articulated mechanism.

Relevant für jeden Roboterarm, 3D-Drucker, jede CNC-Maschine und jeden Gelenkmechanismus.

Forward vs. Inverse Kinematics

Vorwärts- vs. inverse Kinematik

Forward (FK)

Vorwärts (FK)

joint angles → end-effector pose

Gelenkwinkel → Endeffektor-Pose

Usually a straightforward chain of transforms. One unique answer.

Meist eine einfache Kette von Transformationen. Eindeutige Lösung.

Inverse (IK)

Inverse (IK)

end-effector pose → joint angles

Endeffektor-Pose → Gelenkwinkel

Often multiple solutions, sometimes none. This is what a robot controller needs every cycle.

Oft mehrere Lösungen, manchmal keine. Genau das braucht eine Robotersteuerung in jedem Zyklus.

Serial vs. Parallel Manipulators

Serielle vs. parallele Manipulatoren

Serial

Seriell

Joints chained one after another (e.g. a 6-axis arm). Large workspace, but the base motors carry all the following links' weight → slower, less rigid.

Gelenke hintereinander verkettet (z. B. ein 6-Achs-Arm). Großer Arbeitsraum, aber die Motoren an der Basis tragen das Gewicht aller folgenden Glieder → langsamer, weniger steif.

Parallel

Parallel

Multiple limbs connect a fixed base directly to a moving platform. Motors stay fixed at the base → low inertia, very fast and rigid. Delta robots are the classic example.

Mehrere Glieder verbinden eine feste Basis direkt mit einer beweglichen Plattform. Motoren bleiben an der Basis fixiert → geringe Trägheit, sehr schnell und steif. Delta-Roboter sind das klassische Beispiel.

Why Delta Robots?

Warum Delta-Roboter?

  • All 3 motors mounted on the fixed base — the moving arms are light
  • Extremely fast, high acceleration (pick-and-place, sorting)
  • Used in food/pharma packaging, electronics assembly, delta-style 3D printers
  • Alle 3 Motoren sitzen fest an der Basis — die beweglichen Arme sind leicht
  • Extrem schnell, hohe Beschleunigung (Pick-and-Place, Sortieren)
  • Einsatz in Lebensmittel-/Pharmaverpackung, Elektronikmontage, Delta-3D-Druckern

Delta Robot Anatomy

Aufbau des Delta-Roboters

  • Base — fixed, holds 3 rotary motors 120° apart
  • Bicep (upper arm) — one per motor, actively driven
  • Forearm — a passive parallelogram linkage per arm
  • Platform (end-effector) — moves in x, y, z
  • Basis — fest, trägt 3 Drehmotoren im 120°-Abstand
  • Bizeps (Oberarm) — einer pro Motor, aktiv angetrieben
  • Unterarm — ein passives Parallelogramm-Gestänge pro Arm
  • Plattform (Endeffektor) — bewegt sich in x, y, z

Each of the 3 identical arms: a driven bicep + a passive parallelogram forearm that keeps the platform level.

Jeder der 3 identischen Arme: ein angetriebener Bizeps + ein passives Parallelogramm-Unterarm, das die Plattform waagerecht hält.

The Key Simplification

Die entscheidende Vereinfachung

Each forearm is a parallelogram, not a single rod.

Jeder Unterarm ist ein Parallelogramm, keine einzelne Stange.

⇒ the platform's orientation is always parallel to the base.

⇒ Die Orientierung der Plattform ist stets parallel zur Basis.

Only 3 translational DOF remain (x, y, z) — no orientation to solve for. That's what makes the delta IK tractable in closed form.

Es bleiben nur 3 translatorische Freiheitsgrade (x, y, z) — keine Orientierung zu lösen. Genau das macht die Delta-IK geschlossen lösbar.

Problem Statement

Problemstellung

Given a target platform center \(P=(x,y,z)\) (relative to the base center), find the 3 bicep angles \(\theta_1,\theta_2,\theta_3\) that place the platform there.

Gegeben sei ein Ziel-Plattformmittelpunkt \(P=(x,y,z)\) (relativ zum Basismittelpunkt); gesucht sind die 3 Bizepswinkel \(\theta_1,\theta_2,\theta_3\), die die Plattform dorthin bringen.

1 Exploit the 3-fold SymmetryDie 3-fache Symmetrie nutzen

The 3 arms are identical, spaced 120° apart. Solve one arm in its own rotated frame, then reuse the same formula 3× by rotating \(P\) by \(\gamma_i = 0°, 120°, 240°\).

Die 3 Arme sind identisch und im 120°-Abstand angeordnet. Löse einen Arm in seinem eigenen rotierten Bezugssystem und wende dieselbe Formel 3× an, indem \(P\) um \(\gamma_i = 0°, 120°, 240°\) gedreht wird.

Same target \(P\), three shoulders. Rotating \(P\) by \(-\gamma_i\) (equivalently, rotating the frame by \(\gamma_i\)) lets every arm reuse arm 1's formula.

Gleiches Ziel \(P\), drei Schultern. Das Drehen von \(P\) um \(-\gamma_i\) (äquivalent: Drehen des Bezugssystems um \(\gamma_i\)) lässt jeden Arm dieselbe Formel von Arm 1 wiederverwenden.

2 Per-arm Local FrameLokales Bezugssystem pro Arm

Rotate \(P\) into arm \(i\)'s frame and split into radial / tangential / vertical components relative to that arm's shoulder joint:

Drehe \(P\) in das Bezugssystem von Arm \(i\) und zerlege es in radiale/tangentiale/vertikale Komponenten relativ zum Schultergelenk dieses Arms:

\[ W_x = x\sin\gamma_i - y\cos\gamma_i - R_{diff} \]

\[ W_y = x\cos\gamma_i + y\sin\gamma_i \]

\[ W_z = z \]

\((W_x,W_y,W_z)\) = the platform joint position, expressed relative to arm \(i\)'s shoulder.

\((W_x,W_y,W_z)\) = die Position des Plattformgelenks, ausgedrückt relativ zur Schulter von Arm \(i\).

Top view of the base: shoulder \(B\) sits at radius \(R_b\) from center \(O\). \(P\)'s position is measured along \(B\)'s own radial/tangential axes (colored bars = the \(W_x, W_y\) you get from the formula).

Draufsicht der Basis: Schulter \(B\) liegt im Radius \(R_b\) um Mittelpunkt \(O\). Die Position von \(P\) wird entlang der radialen/tangentialen Achsen von \(B\) gemessen (farbige Balken = die \(W_x, W_y\) aus der Formel).

3 Elbow Lives in One PlaneDer Ellbogen liegt in einer Ebene

The bicep only rotates in the vertical plane through its shoulder — so in the local frame the elbow has no tangential component:

Der Bizeps rotiert nur in der Vertikalebene durch seine Schulter — im lokalen Bezugssystem hat der Ellbogen also keine tangentiale Komponente:

\[ E = (L_b\cos\theta_i,\ 0,\ -L_b\sin\theta_i) \]

Side slice through shoulder \(B\) along its radial direction (see inset). The bicep (red) swings only in this plane, reaching \(E\) — its horizontal/vertical legs are exactly \(L_b\cos\theta\) and \(L_b\sin\theta\). The forearm (blue, dashed) then reaches \(W\), which can also shift sideways (tangentially) out of the plane.

Seitenschnitt durch Schulter \(B\) entlang ihrer radialen Richtung (siehe Einschub). Der Bizeps (rot) schwenkt nur in dieser Ebene bis zu \(E\) — seine horizontale/vertikale Komponente sind genau \(L_b\cos\theta\) und \(L_b\sin\theta\). Der Unterarm (blau, gestrichelt) erreicht dann \(W\), das zusätzlich seitlich (tangential) aus der Ebene heraus verschoben sein kann.

4 Apply the Forearm Length ConstraintDie Unterarmlängen-Bedingung anwenden

The forearm is rigid, length \(L_f\): \(|W-E| = L_f\). Since \(|W|^2=W_x^2+W_y^2+W_z^2\) is already known, expanding and collecting the constants as \(K\):

Der Unterarm ist starr mit Länge \(L_f\): \(|W-E| = L_f\). Da \(|W|^2=W_x^2+W_y^2+W_z^2\) bereits bekannt ist, ergibt das Ausmultiplizieren und Zusammenfassen der Konstanten zu \(K\):

\[ K = |W|^2 + L_b^2 - L_f^2 \]

In arm \(i\)'s plane, \(E\) must sit on both circles: distance \(L_b\) from shoulder \(B\), and distance \(L_f\) from \(W\) (adjusted for the sideways offset \(W_y\)). Two circles → up to two crossing points.

In der Ebene von Arm \(i\) muss \(E\) auf beiden Kreisen liegen: Abstand \(L_b\) von Schulter \(B\) und Abstand \(L_f\) von \(W\) (angepasst um den seitlichen Versatz \(W_y\)). Zwei Kreise → bis zu zwei Schnittpunkte.

4 Apply the Forearm Length ConstraintDie Unterarmlängen-Bedingung anwenden (cont.)(Forts.)

Substituting \(E=(L_b\cos\theta_i,\,0,\,-L_b\sin\theta_i)\) turns this into a single equation in \(\theta_i\):

Einsetzen von \(E=(L_b\cos\theta_i,\,0,\,-L_b\sin\theta_i)\) macht daraus eine einzige Gleichung in \(\theta_i\):

\[ K + 2L_b\big(W_z\sin\theta_i - W_x\cos\theta_i\big) = 0 \]

i.e. \( a\sin\theta + b\cos\theta = c \), with \(a=2L_bW_z,\ b=-2L_bW_x,\ c=-K\).

d. h. \( a\sin\theta + b\cos\theta = c \), mit \(a=2L_bW_z,\ b=-2L_bW_x,\ c=-K\).

4 Apply the Forearm Length ConstraintDie Unterarmlängen-Bedingung anwenden — Workspace & Singularities— Arbeitsraum & Singularitäten

  • Same two circles as above. As the target moves toward the workspace edge, they slide apart: 2 crossings (normal) → 1 tangent point (singular) → 0 crossings (unreachable)
  • That's exactly the sign of disc = a²+b²-c² from Step 5
  • Reachable volume overall is also bounded by \(L_b+L_f\) and each joint's angle limits
  • Dieselben zwei Kreise wie oben. Bewegt sich das Ziel zum Rand des Arbeitsraums, driften sie auseinander: 2 Schnittpunkte (normal) → 1 Tangentenpunkt (singulär) → 0 Schnittpunkte (unerreichbar)
  • Das ist genau das Vorzeichen von disc = a²+b²-c² aus Schritt 5
  • Das insgesamt erreichbare Volumen ist zudem durch \(L_b+L_f\) sowie die Winkelgrenzen jedes Gelenks begrenzt

5 Solve with the Half-angle SubstitutionLösung über die Halbwinkel-Substitution

Let \(u=\tan(\theta_i/2)\), so \(\sin\theta=\frac{2u}{1+u^2}\), \(\cos\theta=\frac{1-u^2}{1+u^2}\). The equation becomes quadratic:

Sei \(u=\tan(\theta_i/2)\), also \(\sin\theta=\frac{2u}{1+u^2}\), \(\cos\theta=\frac{1-u^2}{1+u^2}\). Die Gleichung wird quadratisch:

\[ (b+c)\,u^2 - 2a\,u - (b-c) = 0 \]

\[ u = \frac{a \pm \sqrt{a^2+b^2-c^2}}{b+c} \qquad \theta_i = 2\arctan(u) \]

If \(a^2+b^2-c^2 < 0\): the target is unreachable for this arm.

Falls \(a^2+b^2-c^2 < 0\): Das Ziel ist für diesen Arm unerreichbar.

6 Two Solutions — Pick the Physical OneZwei Lösungen — die physikalische wählen

The \(\pm\) gives two elbow configurations. In practice, pick the branch with the smaller \(|\theta_i|\) — the arm that doesn't fold back past vertical (or, for a moving robot, whichever branch is closest to the previous \(\theta_i\), for smooth motion).

Das \(\pm\) liefert zwei Ellbogenkonfigurationen. In der Praxis wählt man den Zweig mit dem kleineren \(|\theta_i|\) — den Arm, der nicht über die Vertikale hinaus zurückklappt (oder, bei einem bewegten Roboter, den Zweig, der dem vorherigen \(\theta_i\) am nächsten liegt, für eine ruckfreie Bewegung).

Same two candidates from Step 4. \(E_2\) keeps the bicep swinging outward and down (physical); \(E_1\) would require it to fold back toward the center — geometrically valid, but not how the arm can actually move.

Dieselben zwei Kandidaten aus Schritt 4. \(E_2\) lässt den Bizeps nach außen und unten schwenken (physikalisch); \(E_1\) würde ein Zurückklappen zur Mitte erfordern — geometrisch gültig, aber nicht so, wie sich der Arm tatsächlich bewegen kann.

7 Repeat for All 3 ArmsFür alle 3 Arme wiederholen

Closed form for each arm \(i\) (\(\gamma_i\) from Step 2, \(a_i,b_i,c_i\) from Step 4):

Geschlossene Form für jeden Arm \(i\) (\(\gamma_i\) aus Schritt 2, \(a_i,b_i,c_i\) aus Schritt 4):

\[ \theta_i = 2\arctan\!\left(\frac{a_i+\sqrt{a_i^2+b_i^2-c_i^2}}{b_i+c_i}\right) \]

\[ \gamma_i = \tfrac{2\pi i}{3},\ \ i=0,1,2 \]

Do this once per arm, using that arm's own \(\gamma_i\), \(a_i\), \(b_i\), \(c_i\) — three independent 1-D solves, no coupling between arms.

Das einmal pro Arm mit dessen eigenem \(\gamma_i\), \(a_i\), \(b_i\), \(c_i\) durchführen — drei unabhängige 1-D-Lösungen, keine Kopplung zwischen den Armen.

7 Repeat for All 3 ArmsFür alle 3 Arme wiederholen (cont.)(Forts.)

Equivalent Python:

Äquivalenter Python-Code:

import math

def delta_ik(x, y, z, Rb, Rp, Lb, Lf):
    Rdiff = Rb - Rp
    theta = [0, 0, 0]
    for i in range(3):
        gamma = math.radians(i * 120)
        Wx = x*math.sin(gamma) - y*math.cos(gamma) - Rdiff
        Wy = x*math.cos(gamma) + y*math.sin(gamma)
        Wz = z
        a = 2*Lb*Wz
        b = -2*Lb*Wx
        c = -(Wx**2 + Wy**2 + Wz**2 + Lb**2 - Lf**2)
        disc = a*a + b*b - c*c
        if disc < 0:
            return None                       # unreachable
        u = (a + math.sqrt(disc)) / (b + c)   # or (a - sqrt(disc))/(b+c)
        theta[i] = 2 * math.atan(u)           # pick physical branch
    return theta

Forward Kinematics: the Reverse Problem

Vorwärtskinematik: das umgekehrte Problem

Given the 3 bicep angles \(\theta_1,\theta_2,\theta_3\) (and \(R_b,R_p,L_b,L_f\)), find the platform center \(P=(x,y,z)\).

Gegeben seien die 3 Bizepswinkel \(\theta_1,\theta_2,\theta_3\) (sowie \(R_b,R_p,L_b,L_f\)); gesucht ist der Plattformmittelpunkt \(P=(x,y,z)\).

Unlike IK, this doesn't split into 3 independent 1-D problems — all 3 arms must agree on the same \(P\), so it takes a different trick: trilateration.

Anders als bei der IK zerfällt dies nicht in 3 unabhängige 1-D-Probleme — alle 3 Arme müssen sich auf denselben Punkt \(P\) einigen. Das erfordert einen anderen Kniff: Trilateration.

1 Each Arm Constrains P to a SphereJeder Arm schränkt P auf eine Kugel ein

\(\theta_i\) is known, so elbow \(E\) is known. Shift it by \(R_p\) (same trick as \(R_{diff}\)) to get a fixed sphere center \(C_i\):

\(\theta_i\) ist bekannt, also auch der Ellbogen \(E\). Verschiebt man ihn um \(R_p\) (derselbe Kniff wie bei \(R_{diff}\)), erhält man einen festen Kugelmittelpunkt \(C_i\):

\[ k_i = R_{diff} + L_b\cos\theta_i \]

\[ h_i = -L_b\sin\theta_i \]

\[ C_i = (k_i\sin\gamma_i,\ -k_i\cos\gamma_i,\ h_i) \]

The forearm is rigid: \(|P-C_i|=L_f\).

Der Unterarm ist starr: \(|P-C_i|=L_f\).

Side slice through shoulder \(B\) (same plane as Step 3 of the IK derivation). \(C_i\) is not the elbow itself, just the elbow nudged by \(R_p\).

Seitenschnitt durch Schulter \(B\) (dieselbe Ebene wie in Schritt 3 der IK-Herleitung). \(C_i\) ist nicht der Ellbogen selbst, sondern der um \(R_p\) verschobene Ellbogen.

2 Trilateration: Three Spheres, One PointTrilateration: drei Kugeln, ein Punkt

Each of the 3 arms gives one sphere (Step 1), all with the same known radius \(L_f\) but a different, arm-specific center \(C_i\):

Jeder der 3 Arme liefert eine Kugel (Schritt 1), alle mit demselben bekannten Radius \(L_f\), aber je einem arm-eigenen Mittelpunkt \(C_i\):

\[ |P-C_i| = L_f \quad (i=0,1,2) \]

\(P\) has to satisfy all three at once — it's the (generically unique, reachable) common intersection point of the three spheres.

\(P\) muss alle drei gleichzeitig erfüllen — er ist der (im Allgemeinen eindeutige, erreichbare) gemeinsame Schnittpunkt der drei Kugeln.

Schematic top view: each dashed circle is one arm's sphere (radius \(L_f\) around its \(C_i\)). \(P\) is where all three overlap.

Schematische Draufsicht: Jeder gestrichelte Kreis ist die Kugel eines Arms (Radius \(L_f\) um sein \(C_i\)). \(P\) liegt dort, wo sich alle drei überlappen.

3 Subtract Pairs → Two PlanesPaare subtrahieren → zwei Ebenen

Three spheres, each with the \(|P|^2\) quadratic term — but that term is the same in every equation. Subtracting arm 0's equation from arm 1's and arm 2's cancels it, leaving two linear equations (planes) in \(x,y,z\):

Drei Kugeln, jede mit dem quadratischen Term \(|P|^2\) — aber dieser Term ist in jeder Gleichung gleich. Zieht man Gleichung 0 von Gleichung 1 und 2 ab, hebt er sich weg, und es bleiben zwei lineare Gleichungen (Ebenen) in \(x,y,z\):

\[ 2(C_i - C_0)\cdot P = |C_i|^2 - |C_0|^2, \qquad i = 1, 2 \]

Two planes in three unknowns aren't enough to pin down \(P\) alone — but they're enough to express \(x\) and \(y\) as linear functions of \(z\).

Zwei Ebenen in drei Unbekannten reichen nicht aus, um \(P\) allein festzulegen — aber sie reichen, um \(x\) und \(y\) als lineare Funktionen von \(z\) auszudrücken.

4 Solve the Linear System, Then the QuadraticDas lineare System lösen, dann das quadratische

Solving the 2×2 linear system from Step 3 gives \(x\) and \(y\) in terms of \(z\):

Löst man das 2×2-lineare System aus Schritt 3, erhält man \(x\) und \(y\) in Abhängigkeit von \(z\):

\[ x = m_x z + n_x \qquad y = m_y z + n_y \]

Substituting both into arm 0's sphere equation \(|P-C_0|=L_f\) leaves a single quadratic — now in \(z\) alone:

Setzt man beide in die Kugelgleichung \(|P-C_0|=L_f\) von Arm 0 ein, bleibt eine einzige quadratische Gleichung — nun allein in \(z\):

\[ A z^2 + B z + C = 0 \]

5 Two Solutions — Pick the Physical OneZwei Lösungen — die physikalische wählen

The quadratic's two roots are two candidate platform heights:

Die beiden Wurzeln der quadratischen Gleichung sind zwei mögliche Plattformhöhen:

\[ z = \frac{-B \pm \sqrt{B^2-4AC}}{2A} \]

Take the lower root (further below the base) — the arms physically hang down, they can't fold the platform back up near the base. Same logic as Step 6 of the IK derivation.

Man wählt die untere Wurzel (weiter unterhalb der Basis) — die Arme hängen physisch nach unten und können die Plattform nicht zurück in Basisnähe klappen. Dieselbe Logik wie in Schritt 6 der IK-Herleitung.

Two \(z\) roots → two heights for the same three arm angles. Only the deeper one is how a real delta robot can actually hang.

Zwei \(z\)-Wurzeln → zwei Höhen für dieselben drei Armwinkel. Nur die tiefere entspricht der Art, wie ein realer Delta-Roboter tatsächlich hängen kann.

6 Assemble the Closed FormDie geschlossene Lösung zusammensetzen

Putting Steps 1–5 together, per arm \(i=0,1,2\), reusing \(k_i,h_i\) from Step 1:

Schritte 1–5 zusammengesetzt, für jeden Arm \(i=0,1,2\), mit \(k_i,h_i\) aus Schritt 1:

\[ C_i = (k_i\sin\gamma_i,\ -k_i\cos\gamma_i,\ h_i) \]

then solve the 2 plane equations (Step 3) for \(x(z), y(z)\), substitute into arm 0's sphere equation, and take the lower root of the resulting quadratic (Steps 4–5):

dann die 2 Ebenengleichungen (Schritt 3) nach \(x(z), y(z)\) lösen, in die Kugelgleichung von Arm 0 einsetzen und die untere Wurzel der resultierenden quadratischen Gleichung nehmen (Schritte 4–5):

\[ z = \frac{-B-\sqrt{B^2-4AC}}{2A} \]

\[ x = m_xz+n_x, \qquad y = m_yz+n_y \]

6 Assemble the Closed FormDie geschlossene Lösung zusammensetzen (cont. 1/2)(Forts. 1/2)

import math

def delta_fk(thetas, Rb, Rp, Lb, Lf):
    Rdiff = Rb - Rp
    C = []
    for i in range(3):
        g = math.radians(i * 120)
        k = Rdiff + Lb * math.cos(thetas[i])
        h = -Lb * math.sin(thetas[i])
        C.append((k * math.sin(g), -k * math.cos(g), h))
    C0, C1, C2 = C
    def sq(p): return p[0]**2 + p[1]**2 + p[2]**2
    S0 = sq(C0)
    # plane(Ci): 2(Ci-C0)*P = |Ci|^2 - |C0|^2
    def plane(Ci):
        Si = sq(Ci)
        return (2*(Ci[0]-C0[0]), 2*(Ci[1]-C0[1]), 2*(Ci[2]-C0[2]), Si - S0)

6 Assemble the Closed FormDie geschlossene Lösung zusammensetzen (cont. 2/2)(Forts. 2/2)

    # solve the 2x2 system for x, y in terms of z
    det = A1 * B2 - A2 * B1
    nx = (E1*B2 - E2*B1) / det
    mx = (-D1*B2 + D2*B1) / det
    ny = (A1*E2 - A2*E1) / det
    my = (-A1*D2 + A2*D1) / det
    px, qx = mx, nx - C0[0]
    py, qy = my, ny - C0[1]
    Aq = px*px + py*py + 1
    Bq = 2*px*qx + 2*py*qy - 2*C0[2]
    Cq = qx*qx + qy*qy + C0[2]**2 - Lf**2
    disc = Bq*Bq - 4*Aq*Cq
    if disc < 0:
        return None                  # thetas don't match a valid pose
    z = (-Bq - math.sqrt(disc)) / (2*Aq)   # lower root = physical
    x = mx * z + nx
    y = my * z + ny
    return (x, y, z)

Summary

Zusammenfassung

  • Kinematics: FK is usually easy, IK is the interesting problem — for delta robots it flips: IK is closed-form per arm, FK needs trilateration across all 3
  • Delta robots trade 6-DOF generality for 3-DOF speed & rigidity
  • Parallelogram forearms → orientation-free platform → closed-form IK
  • Per-arm symmetry + one quadratic equation solves the whole robot (either direction)
  • Kinematik: FK ist meist einfach, IK das interessante Problem — bei Delta-Robotern kehrt sich das um: IK ist pro Arm geschlossen lösbar, FK braucht Trilateration über alle 3
  • Delta-Roboter tauschen 6-DOF-Allgemeinheit gegen 3-DOF-Geschwindigkeit & Steifigkeit
  • Parallelogramm-Unterarme → orientierungsfreie Plattform → geschlossene IK-Lösung
  • Symmetrie pro Arm + eine quadratische Gleichung löst den gesamten Roboter (in beide Richtungen)
